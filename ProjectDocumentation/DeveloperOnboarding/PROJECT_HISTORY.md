## PROJECT OVERVIEW

### What We're Building
An XR bubble interaction system where users can touch and manipulate floating bubbles in 3D space using hand tracking or controllers. The core innovation is the **wave mathematics system** that drives bubble positioning, breathing animations, and interaction responses using real mathematical wave calculations.

### Why It Matters
Traditional XR UI is flat and unnatural. Our wave-driven bubble system creates organic, mathematically beautiful interactions that feel natural in 3D space. The wave mathematics aren't just visual - they're the core interaction paradigm.

## CORE TECHNOLOGY

### Wave Mathematics System ⭐ **CRITICAL COMPONENT**

**Why It's Important**: This is the heart of our interaction system. The wave mathematics aren't decorative - they're functional.

**What It Does**:
- **Bubble Positioning**: Uses wave interference patterns to arrange bubbles in mathematically pleasing formations
- **Breathing Animation**: Natural breathing rhythm (0.25 Hz) using sine wave calculations
- **Interaction Response**: Touch creates wave propagation that affects nearby bubbles
- **Spatial Harmony**: Golden ratio and Fibonacci sequences for natural arrangement

**Key Files**:
- `UnityProject/Assets/XRBubbleLibrary/Mathematics/WaveParameters.cs` - Core mathematical parameters
- `UnityProject/Assets/XRBubbleLibrary/Mathematics/WavePatternGenerator.cs` - Wave calculation engine
- `UnityProject/Assets/XRBubbleLibrary/Mathematics/CymaticsController.cs` - Visual wave patterns

**Mathematical Principles**:
- **Wave Interference**: Multiple wave sources create complex patterns
- **Harmonic Resonance**: Bubbles respond to mathematical frequency relationships
- **Cymatics Integration**: Sound waves create visual patterns (like sand on vibrating plates)
- **Performance Optimization**: Real-time calculations optimized for Quest 3 mobile hardware

### XR Interaction System

**Purpose**: Translate hand/controller input into wave mathematics  
**Approach**: XR Interaction Toolkit integration with custom wave response

**Key Components**:
- Hand tracking detection

## ARCHITECTURAL DECISIONS

### Why We Chose This Architecture

**Linear Assembly Dependencies**:
```
Core → Mathematics → Physics → UI → Interactions → AI
```

**Rationale**: Prevents circular dependencies that caused original compilation errors

**Core Assembly**: Basic interfaces and shared data structures  
**Mathematics Assembly**: All wave calculations (the heart of the system)  
**Physics Assembly**: Bubble physics driven by wave mathematics  
**UI Assembly**: Visual representation of mathematical calculations  
**Interactions Assembly**: XR input that triggers wave responses  
**AI Assembly**: Advanced features (deferred to Phase 2)

### Why We Preserved Wave Mathematics

**Original Temptation**: Simplify by removing complex mathematics  
**Decision**: Keep wave mathematics as core feature  
**Rationale**: The mathematics ARE the innovation - removing them makes this just another bubble UI

**CEO Quote**: "The wave mathematics is extremely important as that's the key of the interaction. This is a huge part of the process and intended product."
